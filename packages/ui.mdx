`@lindaui/ui` is the core React component layer of the Linda UI design system. It provides 71 components built on top of HeroUI v3 (react-aria-components), each exported from its own dedicated entry point. Where wrapping a HeroUI primitive would expose an awkward RAC composition API, Linda UI replaces it with a flat prop surface — no compound children, no leaked internal types. Components that are thin pass-throughs re-export the HeroUI compound parts directly.

## Installation

```bash
pnpm add @lindaui/ui @lindaui/tokens
```

Then import the CSS bundle once at your app root:

```tsx
import "@lindaui/tokens/css";
```

Your project must provide the following:

| Package | Version |
|---|---|

## Import convention

Every component lives at its own subpath. There is **no barrel export** from the package root — always import from the specific entry point:

```tsx
import { Button }      from "@lindaui/ui/button";
import { Input }       from "@lindaui/ui/input";
import { Dialog }      from "@lindaui/ui/dialog";
import { useDisclosure } from "@lindaui/ui/use-disclosure";
```

## Global conventions

Read these before building — they prevent the most common integration mistakes.

<Warning>
  **Controls emit `onChange`, not `onValueChange`.** `Checkbox`, `Radio`,
  `Switch` give `onChange(isSelected: boolean)`; `Slider` gives
  `onChange(value: number)`. There is no `onValueChange` anywhere in the
  Linda UI wrapper API.
</Warning>

- **`Select` and `Combobox`** take `items={[{ value, label }]}` and emit `onChange(value: string)`. There is no `SelectItem` component — pass the flat array.
- **`Dialog`** is always controlled: `open` + `onClose`. There is no `ModalContent`.
- **`Menu`**: the `trigger` prop accepts the button *content* (label, icon, ReactNode) — not a `<button>` element. Passing a `<button>` nests buttons and breaks accessibility.
- **`Button`** uses `variant`, not `color`. Allowed values: `primary | secondary | ghost | danger | link`.
- **Field components (`Input`, `Textarea`, `NumberInput`, `SearchField`) do not accept `className`**. It is a TypeScript error on purpose — any class on the RAC wrapper never reaches the underlying `<input>`. Re-skin all fields at once by overriding the `--field-*` CSS custom properties: `--field-background`, `--field-foreground`, `--field-placeholder`, `--field-border`, `--field-border-hover`, `--field-border-focus`, `--field-focus`, `--field-radius`.
- Interactive components are `"use client"` — they work as leaf client components inside RSC apps.

## Entry points

### Forms & inputs

| Entry point | Description |
|---|---|
| `@lindaui/ui/button` | Action button. `variant: primary \| secondary \| ghost \| danger \| link`. Uses `onPress` (RAC), `isDisabled`, `isPending`. |
| `@lindaui/ui/input` | Single-line text field. Flat `label`, `placeholder`, `onChange(value: string)`, `isInvalid`, `isRequired`. No `className`. |
| `@lindaui/ui/textarea` | Multiline text field. Same flat API as `input`. |
| `@lindaui/ui/number-input` | Numeric field. `onChange(value: number)`, `minValue`, `maxValue`, `step`, `formatOptions`. |
| `@lindaui/ui/search-field` | Text field with built-in magnifier icon and clear button. Emits the string only — do matching with `@lindaui/ui/search`. No `className`. |
| `@lindaui/ui/input-group` | Layout wrapper for grouping an input with addons or action buttons. |
| `@lindaui/ui/otp-input` | One-time password input. Re-exports the `input-otp` API. `maxLength`, `onChange(value: string)`. |
| `@lindaui/ui/select` | Dropdown selector. `items: { value, label }[]`, `value?`, `defaultValue?`, `onChange(value: string)`. Pass `aria-label` when no visible label is shown. |
| `@lindaui/ui/combobox` | Typeahead select. Adds `onInputChange(value: string)` to the select API. |
| `@lindaui/ui/field` | React Hook Form adapter. Render-prop: `({ field, invalid }) => ReactNode`. Shows `hint` or the RHF error message. |

### Controls

| Entry point | Description |
|---|---|
| `@lindaui/ui/checkbox` | Single checkbox. `isSelected`, `onChange(isSelected: boolean)`. `children` is the label. |
| `@lindaui/ui/checkbox-group` | Grouped checkboxes. `value: string[]`, `onChange(string[])`. |
| `@lindaui/ui/radio` | Single radio button. `value` string, used inside `radio-group`. |
| `@lindaui/ui/radio-group` | Radio button group. `value`, `onChange(value: string)`. |
| `@lindaui/ui/switch` | Toggle switch. `isSelected`, `onChange(isSelected: boolean)`. `children` is the label. |
| `@lindaui/ui/switch-group` | Layout-only grouping wrapper for switches. No `value`/`onChange` of its own. |
| `@lindaui/ui/slider` | Range slider. `value`, `onChange(value: number)`, `minValue`, `maxValue`, `step`. `children` is the label. |
| `@lindaui/ui/toggle-button` | Single pressable toggle. |
| `@lindaui/ui/toggle-button-group` | Group of toggle buttons. `selectionMode: "single" \| "multiple"`, `onSelectionChange(Set<Key>)`. |
| `@lindaui/ui/close-button` | Icon close/dismiss button. `onPress`. |
