Linda UI is a production-ready design system built for React 19. It gives you a layered toolkit — brand tokens, accessible UI primitives, and fully composed section blocks — so you can move from an empty project to a polished, accessible interface without hand-rolling every pattern. Every visual decision flows through CSS custom properties, which means you can re-skin the entire system at runtime without touching a single component.

## Architecture

Linda UI is organized as three discrete packages that build on each other in a pyramid:

```
@lindaui/blocks   ← 22 composed sections (LoginForm, ListPane, charts …)
     ↑
@lindaui/ui       ← 71 accessible component primitives
     ↑
@lindaui/tokens   ← brand CSS (OKLCH tokens, HeroUI v3 styles, Tailwind v4 utilities)
```

**`@lindaui/tokens`** is the foundation. It ships a single CSS bundle that wires together HeroUI v3's base styles, the teal OKLCH brand palette, and the Tailwind v4 utilities that components use — all driven by canonical CSS custom properties.

**`@lindaui/ui`** wraps HeroUI v3 and react-aria-components behind stable, per-component entry points. Each component exposes its own ergonomic API rather than leaking the underlying react-aria composition model, so your imports stay clean and your types stay predictable.

**`@lindaui/blocks`** assembles UI primitives from `@lindaui/ui` into ready-to-use sections — authentication flows, master-detail panels, charts, and more. You import a block, wire up your data and handlers, and get a complete, accessible section in return.

## Key features

- **Accessible by default** — components are built on react-aria-components and meet WCAG 2.1 AA out of the box
- **Brand tokens** — a single OKLCH teal palette drives every surface, border, and focus ring; override a handful of CSS variables to match your brand
- **Composable blocks** — 22 pre-assembled sections reduce repetitive layout work without locking you into a framework
- **TypeScript-first** — every package ships full type definitions; component APIs are stable and won't leak internal HeroUI types
- **Mobile-first** — all components and blocks are responsive, tested at small viewports before large ones
- **Dark mode ready** — add a `.dark` class to any ancestor element and the entire palette switches automatically
