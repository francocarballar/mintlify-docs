---

Linda UI ships as three focused packages that you can install together in one command. This guide walks you through installation, wiring up the CSS bundle at your app root, and rendering your first component and block. By the end you'll have a working button and a fully functional login form.

<Note>
  Linda UI requires the following peer dependencies in your project: `react >=19`, `react-dom >=19`, and `react-hook-form >=7`. Blocks that render charts also need `recharts >=3`, and all blocks use icons from `lucide-react >=0.4`. Install any that are missing alongside the Linda UI packages.
</Note>

<Steps>

### Install the packages

Install all three Linda UI packages and their peer dependencies:

<CodeGroup>

```bash npm
npm install @lindaui/tokens @lindaui/ui @lindaui/blocks react react-dom react-hook-form lucide-react
```

```bash yarn
yarn add @lindaui/tokens @lindaui/ui @lindaui/blocks react react-dom react-hook-form lucide-react
```

```bash pnpm
pnpm add @lindaui/tokens @lindaui/ui @lindaui/blocks react react-dom react-hook-form lucide-react
```

</CodeGroup>

### Import the CSS bundle at your app root

Import `@lindaui/tokens/css` **once** at the top of your application entry point. This single import loads HeroUI v3 base styles, the OKLCH brand tokens, and all Tailwind v4 utilities the components rely on.

For a Next.js App Router project, add the import to your root layout:

```tsx app/layout.tsx
import "@lindaui/tokens/css";
import type { ReactNode } from "react";

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

<Tip>
  To enable dark mode, add the `.dark` class to any ancestor element — for example `<html className="dark">` for a site-wide dark theme, or `<div className="dark">` to scope it to a section. Remove the class to return to light mode. No extra configuration is required.
</Tip>

### Render your first component

Import a component from its dedicated entry point and use it in any client or server component. Here is a simple button example:

```tsx app/page.tsx
import { Button } from "@lindaui/ui/button";

export default function Page() {
  return (
    <main>
      <Button variant="primary" onPress={() => console.log("clicked")}>
        Get started
      </Button>
    </main>
  );
}
```

`Button` accepts a `variant` prop (`"primary"`, `"secondary"`, `"ghost"`, `"danger"`, or `"link"`) along with all standard react-aria button props such as `isDisabled` and `onPress`.

### Add your first block

Blocks are pre-assembled sections that manage their own internal state. You provide data and event handlers; the block handles rendering and accessibility. Here is the `LoginForm` block wired to an async submit handler:

```tsx app/login/page.tsx
"use client";

import { LoginForm, type LoginValues } from "@lindaui/blocks/login-form";

export default function LoginPage() {
  async function handleSubmit(values: LoginValues) {
    // values: { email: string; password: string }
    const response = await fetch("/api/auth/login", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(values),
    });

    if (!response.ok) {
      throw new Error("Login failed");
    }
  }

  return (
    <div className="flex min-h-screen items-center justify-center">
      <LoginForm
        onSubmit={handleSubmit}
        title="Sign in to your account"
        submitLabel="Sign in"
      />
    </div>
  );
}
```

`LoginForm` renders email and password fields, manages form state with react-hook-form internally, and calls your `onSubmit` handler with typed `LoginValues`. Pass `loading={true}` to disable the submit button during async work, and pass an `error` string to surface server-side errors inline.

</Steps>

You now have a working Linda UI setup. Next, explore [theming](/theming) to learn how to override CSS variables and apply your own brand.
